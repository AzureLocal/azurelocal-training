# Module 01: Planning & Sizing

**Level:** L200 | **Duration:** 2 hours | **Prerequisites:** Module 00 | **Hands-on:** Presentation + design exercise

!!! note "Module Status"
    Framework module — content in development.

## Learning Objectives

- Plan an Azure Local deployment given workload, capacity, and connectivity requirements
- Make appropriate hardware, network, and storage design decisions
- Choose between connected and disconnected architectures
- Plan multi-rack deployments
- Calculate cluster capacity for storage, compute, and growth

## Topics

1. Connected vs. Disconnected Architectures — when to choose each
2. Multi-Rack Deployments — design considerations and constraints
3. Workload Requirements Analysis — sizing for VMs, AKS, AVD
4. Hardware Sizing — node count, CPU, memory, storage
5. Network Design Decisions — Network ATC intents, fabric topology
6. Storage Design Decisions — disk tiers, fault tolerance, capacity planning
7. Identity, Subscription, and Region Planning

## Hands-on

**Design exercise:** Given a customer scenario (sizing requirements, workloads, constraints), produce a sizing recommendation and design diagram. Walk through the design as a group.
