# Module 18: Migration

**Level:** L300 | **Duration:** 3 hours | **Prerequisites:** Modules 02, 05 | **Hands-on:** Lab/Demo

!!! note "Module Status"
    Framework module — content in development.

## Learning Objectives

- Plan a migration from VMware vSphere to Azure Local
- Plan a migration from traditional Hyper-V (Windows Server) to Azure Local
- Use Azure Migrate to assess and migrate workloads
- Handle common migration scenarios and gotchas
- Validate post-migration

## Topics

1. Migration Planning — assessment, prioritization
2. **VMware → Azure Local** — discovery, dependency mapping, conversion approach
3. **Hyper-V → Azure Local** — preserving VMs and configuration
4. Azure Migrate Integration
5. Migration Tools — what works, what doesn't
6. Network and IP Address Migration
7. Storage Migration Patterns
8. Post-Migration Validation
9. Common Gotchas and How to Handle Them

## Hands-on

**Lab/Demo:** Walk through a VMware → Azure Local migration. Tools-only walkthrough if hardware not available — full lab if a vSphere lab environment is provided.
