# Status

Tracks the current state of each training module and provides a home for standup/sync notes.

## Contents

| File / Folder | Purpose |
|---------------|---------|
| [`module-status.md`](module-status.md) | One-row-per-module status board — content, IaC, slides, review |
| [`standups/`](standups/) | Running log of sync notes, decisions, and blockers |

## How to use

- Update `module-status.md` whenever a module phase changes (draft → review → published).
- Drop a standup note in `standups/` after each working session that surfaces a decision or blocker worth preserving.
- This folder is the PM view. For strategic planning, see [`../training-platform-plan.md`](../training-platform-plan.md).
